## Key Decisions
- 进度条改用击杀百分比而非波次索引，因为 waveIndex 在僵尸生成前就已递增导致进度超前
- SVG 纹理作为 `this.add.image()` 使用，有 fallback 到程序化图形绘制
- 自动收集阳光开关放在 SettingsScene 底部（costMultiplier 之后、返回按钮之前）
- 第1波延迟过大（20-25秒）确认为 bug，修复方案：1-1教程关8秒（给新手种1-2棵植物），1-2为10秒（引入向日葵需更多阳光时间），1-10最终关仅6秒（挑战性）；后续波次逐步缩短至4-5秒

## Critical Context
- **Phaser 3.80** 使用 `Phaser.Scale.FIT` + `CENTER_BOTH`，canvas CSS 定位在 (-12, 132.5)，实际 1024x600
- 波次逻辑：`startNextWave()` 在 `create()` 中被调用，设置 `zombieSpawnTimer = wave.delay`，`update()` 中每帧递减 delta，归零时 spawn 一个僵尸，然后下一只 800-1200ms 间隔
- **重要**：当 `waveZombies.length === 0` 且 `waveIndex < waves.length` 且 `aliveZombies <= 2` 时立即启动下一波——这意味着新 delay 值是"前一波僵尸清到≤2只后 N 秒下一波开始出怪"，而非绝对时间线
- Playwright 测试证实：在无玩家操作（不种植植物）情况下，level 1-1 首只僵尸在 ~8秒 出现，随后每 800-1200ms 生成下一只，波次推进逻辑按预期工作
